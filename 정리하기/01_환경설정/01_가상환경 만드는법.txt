/ 01_파이썬3 에서 가상환경 /

파이썬 가상환경은 파이썬 프로젝트를 진행할 때 독립된 환경을 만들어 준다.

/ 가상환경이 필요한 이유 /

예를 들어 파이썬 개발자 2개의 프로젝트를 개발하고 관리한다고 가정할시에 각각의 프로젝트에서 사용할 장고의 버전이 다를 가능성이 존재한다. 그 때 만들어서 사용하면 되는 것이 파이썬 가상환경이다. (각각의 프로젝트가 서로 다른 버전에서만 호환될 수 있기 때문에, 오히려 구버전을 활용해야 할 가능성도 있다.)

/ 요약 /

Ex) 
  ------------------------------------------------
  프로젝트 1번과 2번이 있다고 가정을 할 때,

  프로젝트 1번 (django 3.1)/ 프로젝트 2번(django 4.0)
  ------------------------------------------------

프로젝트 1번과 2번에서 개발에 사용해야 할 장고의 버전이 다를 경우, 이 때 파이썬에서 제공하는 가상환경을 만들어서 사용하면 된다.


/ 사용법 or 만드는 법 /

-- window(key) + R 

윈도우에 내장되어 있는 프로그램을 빠르게 실행하는 화면을 출력


-- cmd 

명령프롬프트를 출력한다 . 


-- c:\Users>

현재 명령프롬프트의 위치를 알려준다. 


-- cd 

Ex) 
  c:\Users> cd/  /* 입력시 C드라이브로 위치 이동 */
  c:\>  

  cd:\Users> cd c:\users\pc /* 입력시 Users/Pc로 위치 이동 */
  C:\Users\PC>

cd를 입력하고 내가 원하는 위치로 이동을 한다


-- mkdir

새로운 폴더를 만들어주는 명령어이다.

Ex)
  c:\>mkdir venvs
  c:\>cd venvs

새로운 디렉터리를 만들고 그 디렉터리로 진입한다.


-- python -m venv mysite

파이썬에서 기본적으로 제공하는 모듈에서 사용가능한 가상환경을 만드는 명령어 이다.

Ex)
  c:\venvs>python -m venv mysite

  python 에서/  -m 모듈이름 / venv(virtual environments) / mysite 파일이름 

아무런 반응이 없지만 venvs에 mysite라는 디텍터리와 가상환경을 구축하는 파일들이 만들어 진다.


/ 가상환경 실행해보기 /

가상환경을 만들었기 때문에 이제 실행을 해서 어떻게 구현되는지 확인해보자 

Ex) 
    C:\venvs>cd C:\venvs\mysite\Scripts 
    C:\venvs\mysite\Scripts> activate
    (mysite) C:\venvs\mysite\Scripts>

가상환경이 만들어진 폴더안으로 먼저 진입을 하여,
activate 명령어를 입력하면 가상환경에 진입한 것을 확인할 수 있다.

가상환경에서 나가고 싶다면, (deactivate) 명령어를 입력하면 된다.

Ex)
    (mysite) C:\venvs\mysite\Scripts> deactivate
    C:\venvs\mysite\Scripts>
